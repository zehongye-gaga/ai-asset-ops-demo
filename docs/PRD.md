# AI 资产运营系统 Demo PRD

## 要解决的问题

集团已有智能体、应用、Skills、知识库和 MCP，但缺少统一目录、运营视图和资产晋级入口。本 Demo 用一条可操作链路证明：资产能统一查看、按范围筛选、由个人提交晋级，并在管理员审批后同步影响列表、待办和大屏指标。

## 使用者

- 个人：查看和管理自己的资产，提交团队晋级申请。
- 团队管理员：查看本团队资产，处理成员晋级申请。
- 系统管理员：查看全量资产，处理通用级晋级并关注运行健康度。

## Demo 范围

- 全局、团队、个人三种视角的运营概览。
- 五类资产统一列表、范围筛选、搜索和详情。
- 个人新建资产、提交晋级申请。
- 团队管理员审批后，资产范围和运营数据同步更新。
- InsForge 持久化资产、审批和动态事件。

## 暂不实现

- Dify、OpenCode、Ccoder 等真实上游同步。
- 生产级身份认证、组织权限和审计留存。
- 知识库文件解析、Skill/MCP 真正的版本回滚。
- 消息队列和定时计算任务。

## 验收标准

1. 页面可以在三种角色和三种视角间切换，数据与操作按钮随之变化。
2. 五类资产都有真实 InsForge 数据，搜索和范围筛选有效。
3. 新建资产刷新页面后仍存在。
4. 晋级申请能进入待办，管理员审批后资产范围发生变化。
5. Rainbond 中四个 InsForge 后端组件和前端组件均为运行状态。
6. 真实浏览器完成大屏、列表、新建、审批和详情操作。
